Section 1: Advanced C# & Web API Core Concepts

1. In modern ASP.NET Core (e.g., .NET 6/8), what is the primary architectural consequence of using ConfigureAwait(false) in your API controllers?
  • A) It prevents deadlocks by forcing the continuation to run on the original synchronization context.
  • B) It has no significant impact on preventing deadlocks because ASP.NET Core does not have a SynchronizationContext.
  • C) It significantly increases CPU overhead due to rapid context switching.
  • D) It automatically disposes of the HttpRequest scope after the await completes.
View Answer & Explanation

Answer: B

Unlike legacy ASP.NET (which used `AspNetSynchronizationContext`), ASP.NET Core removed the SynchronizationContext entirely to improve performance. Therefore, `ConfigureAwait(false)` is largely redundant in ASP.NET Core application code for avoiding deadlocks, though it's still best practice in class libraries to support legacy consumers.

2. Which interface provides covariance in C# generics, allowing you to cast an IEnumerable<Derived> to an IEnumerable<Base>?
  • A) IEnumerable<in T>
  • B) IEnumerable<out T>
  • C) ICollection<T>
  • D) IList<T>
View Answer & Explanation

Answer: B

The out keyword specifies that the type parameter is covariant (used only as a return type). in specifies contravariance (used only as input). IEnumerable<T> is defined as IEnumerable<out T>.

3. When designing a high-performance Web API that needs to parse large blocks of text without allocating objects on the heap, which C# feature should you use?
  • A) StringBuilder
  • B) string.Intern
  • C) Span<T> and ReadOnlySpan<T>
  • D) MemoryStream
View Answer & Explanation

Answer: C

Span<T> is a ref struct that provides type-safe, memory-safe access to contiguous regions of memory (stack, heap, or unmanaged) without requiring heap allocations, drastically reducing Garbage Collection pressure.

4. What is the execution order of filters in the ASP.NET Core Web API pipeline?
  • A) Authorization, Action, Resource, Exception, Result
  • B) Resource, Authorization, Action, Result, Exception
  • C) Authorization, Resource, Action, Exception, Result
  • D) Action, Authorization, Resource, Result, Exception
View Answer & Explanation

Answer: C

The correct pipeline order is: 1. Authorization filters, 2. Resource filters, 3. Action filters, 4. Exception filters, 5. Result filters. (Note: Exception filters run only if an unhandled exception occurs before the result is executed).

5. You have an endpoint that queries a database using Entity Framework Core. You want to avoid tracking the entities in the DbContext to improve read performance. What should you use?
  • A) .AsNoTracking()
  • B) .AsQueryable()
  • C) .Include()
  • D) ChangeTracker.Clear()
View Answer & Explanation

Answer: A

AsNoTracking() tells EF Core not to set up change tracking for the returned entities, saving memory and CPU cycles for read-only queries.

6. In a DI container, you register a service using AddTransient that implements IDisposable. Who is responsible for calling Dispose() on this service?
  • A) The developer must manually cast and call Dispose in the controller.
  • B) The Garbage Collector will eventually finalize it, bypassing Dispose.
  • C) The DI container (Service Provider) automatically tracks and disposes of it at the end of the HTTP request scope.
  • D) Transient services cannot implement IDisposable.
View Answer & Explanation

Answer: C

The built-in IoC container automatically tracks objects it creates that implement IDisposable. When the IServiceScope is disposed (at the end of the HTTP request), all scoped and transient services created within that scope are disposed.

7. When implementing the producer-consumer pattern in a high-throughput API, which modern thread-safe .NET construct is preferred over BlockingCollection?
  • A) ConcurrentQueue<T>
  • B) System.Threading.Channels
  • C) SemaphoreSlim
  • D) Mutex
View Answer & Explanation

Answer: B

System.Threading.Channels is an advanced, highly optimized, asynchronous abstraction for passing data between producers and consumers asynchronously, performing much better than the older, synchronous BlockingCollection.

8. What is the fundamental difference between returning a Task<T> versus a ValueTask<T> from an API service method?
  • A) ValueTask<T> can run on multiple threads simultaneously.
  • B) Task<T> is a value type, whereas ValueTask<T> is a reference type.
  • C) ValueTask<T> avoids heap allocations if the asynchronous operation completes synchronously.
  • D) There is no difference; they compile to the exact same IL.
View Answer & Explanation

Answer: C

Task is a class (reference type), meaning returning it allocates on the heap. ValueTask is a struct (value type). If a method frequently completes synchronously (e.g., retrieving data from a cache), returning ValueTask prevents unnecessary heap allocations.

9. How do you correctly configure ASP.NET Core middleware to permanently short-circuit the request pipeline (preventing subsequent middleware from executing)?
  • A) Use app.Use() and do not call next() or next.Invoke().
  • B) Call next.Stop() inside the middleware delegate.
  • C) Throw a PipelineStopException.
  • D) Use app.Continue(false).
View Answer & Explanation

Answer: A

Middleware is chained. By defining middleware with app.Use (or app.Run) and intentionally omitting the call to the next delegate, the pipeline short-circuits and begins processing the response back up the chain.

10. In C# 9+, what is the functional advantage of using a record class over a standard class for DTOs (Data Transfer Objects)?
  • A) Records bypass Garbage Collection entirely.
  • B) Records provide built-in value-based equality and non-destructive mutation via the 'with' expression.
  • C) Records automatically encrypt sensitive properties during JSON serialization.
  • D) Records can only contain primitive data types.
View Answer & Explanation

Answer: B

Records automatically synthesize methods for value equality (like Equals and GetHashCode based on properties) and support the with keyword for immutable copying, making them ideal for DTOs.

11. What problem does injecting IHttpClientFactory solve compared to instantiating a new HttpClient() manually in every request?
  • A) It prevents thread pool starvation.
  • B) It bypasses SSL certificate validation.
  • C) It solves socket exhaustion and manages DNS refresh lifecycles automatically.
  • D) It increases the maximum payload size allowed by Kestrel.
View Answer & Explanation

Answer: C

Creating a new `HttpClient` per request can lead to socket exhaustion (sockets remaining in `TIME_WAIT` state). Conversely, using a single static `HttpClient` fails to respect DNS changes. `IHttpClientFactory` solves both by pooling and rotating `HttpMessageHandler` instances.

12. You are managing state changes in EF Core. What happens if a concurrency conflict occurs (e.g., two users editing the same record)?
  • A) EF Core automatically overwrites the data with the last request (Last-in Wins).
  • B) EF Core throws an InvalidOperationException during mapping.
  • C) EF Core throws a DbUpdateConcurrencyException if a concurrency token is configured.
  • D) The database engine completely locks the table until restart.
View Answer & Explanation

Answer: C

If you configure a property (like a RowVersion or a specific field) as a concurrency token, EF Core includes it in the `WHERE` clause of the `UPDATE` statement. If 0 rows are affected, it throws `DbUpdateConcurrencyException` to allow your API to handle the conflict.

13. In a high-traffic Web API, a user initiates a long-running data export but closes their browser before it finishes. How do you prevent your API from wasting resources finishing the task?
  • A) Implement a polling mechanism in JavaScript.
  • B) Use GC.Collect() proactively.
  • C) Inject and utilize a CancellationToken in the controller action and pass it to async methods.
  • D) Set a tight timeout on the Kestrel server.
View Answer & Explanation

Answer: C

ASP.NET Core binds CancellationToken to the HTTP request connection. If the client disconnects, the token is cancelled. Passing this token down to EF Core or external HTTP calls allows the operation to be safely aborted, freeing up threads.

14. What is the difference between IOptions<T> and IOptionsSnapshot<T> when injecting configuration data?
  • A) IOptions is registered as Transient; IOptionsSnapshot is Singleton.
  • B) IOptions cannot read from appsettings.json; IOptionsSnapshot can.
  • C) IOptions reads data once on startup (Singleton); IOptionsSnapshot re-reads data per request if the underlying file changes (Scoped).
  • D) There is no difference; they are aliases.
View Answer & Explanation

Answer: C

IOptions is a singleton and will not reflect changes made to `appsettings.json` while the app is running. IOptionsSnapshot is scoped and will pick up configuration changes on the next HTTP request.

15. Why might a developer choose Minimal APIs over MVC Controllers in .NET 6+?
  • A) Minimal APIs bypass security middleware, making them faster.
  • B) Minimal APIs offer less overhead by avoiding controller reflection and heavy MVC routing features, yielding slightly higher throughput for microservices.

Minimal APIs provide a more direct, lower-overhead routing model. They avoid the memory and CPU overhead of discovering and instantiating Controller classes via reflection, resulting in better raw performance for simple microservices.

16. When creating a custom exception handling middleware, what is the best format to return HTTP API errors to the client to comply with RFC 7807?

  • B) A custom XML schema defined by your team.
  • C) Returning a ProblemDetails object.
  • D) Returning HTTP 200 OK with a JSON object containing {"error": true}.

Answer: C

ASP.NET Core has built-in support for ProblemDetails, which is the standardized (RFC 7807) machine-readable format for specifying errors in HTTP API responses (containing properties like type, title, status, and detail).

17. What causes "Thread Pool Starvation" in a .NET Web API application?
  • A) Having too many background worker services running.
  • B) Synchronously blocking asynchronous code using .Result or .Wait() on Tasks.
  • C) Configuring Dependency Injection incorrectly.
  • D) Using the async keyword on a method that doesn't use await.
View Answer & Explanation

Answer: B

Calling `.Result` or `.Wait()` blocks the current thread until the async operation completes. If under load, the thread pool runs out of worker threads because they are all blocked waiting for async tasks to complete, causing the application to hang (starvation).

18. You are streaming a large sequence of database records to the client without loading them all into RAM at once. What return type should you use in C# 8.0+?
  • A) List<Task>
  • B) IEnumerable<Task<T>>
  • C) IAsyncEnumerable<T>
  • D) IQueryable<T>

Answer: C

IAsyncEnumerable<T> (often used with yield return and await foreach) allows you to stream data asynchronously. The API will stream the JSON array to the client as records are fetched, keeping memory footprint low.

19. In the context of JWT Authentication, where should the application store the secret key used to sign the tokens?
  • A) Hardcoded in the Program.cs file.
  • B) In the payload of the JWT itself.
  • C) Securely outside of source control, such as in Azure Key Vault or AWS Secrets Manager.
  • D) Inside a public static class variable.
View Answer & Explanation

Answer: C

Symmetric signing keys must be kept strictly confidential. Storing them in code or public repositories is a critical security vulnerability. They should be retrieved at runtime from secure environment variables or dedicated secret management services.

20. What is the Large Object Heap (LOH) in .NET, and why is it a concern for high-performance Web APIs?
  • A) It stores all value types, causing stack overflow exceptions.
  • B) It stores objects larger than ~85,000 bytes. The LOH is not compacted by default during Garbage Collection, which can lead to memory fragmentation.
  • C) It is the memory segment dedicated strictly to database connection strings.
  • D) It is an obsolete feature removed in .NET 6.
View Answer & Explanation

Answer: B

Allocating large objects (like massive byte arrays or huge strings over 85KB) places them on the LOH. Because moving large chunks of memory is expensive, the GC doesn't normally compact the LOH, which can result in fragmented memory and eventual OutOfMemory exceptions in high-throughput scenarios.
